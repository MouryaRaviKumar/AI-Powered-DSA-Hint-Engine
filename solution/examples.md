## Example 1: Two Sum (Easy)

### 1. Student Request
* **Problem**: Given an array of integers `nums` and an integer `target`, return indices of the two numbers such that they add up to `target`.
* **Language**: Python
* **Student's Code**:
```python
def twoSum(nums, target):
    for i in range(len(nums)):
        for j in range(i + 1, len(nums)):
            if nums[i] + nums[j] == target:
                return [i, j]
```

### 2. Progressive Hint Responses

#### Tier 1: Conceptual Hint (`POST /hint/1`)
* **Thought Process**: Student is using a brute-force approach with nested loops. Time complexity is O(N^2). We need to guide them toward a single pass using a hash map to hit O(N) time.
* **Hint**: You are trading time for memory. Nested loops run in O(N^2) time. Think about how you can check if the "complement" (target - current_number) has already been seen in O(1) time.
* **Conceptual Explanation**: Hash maps provide average O(1) lookups. Storing values as you iterate lets you find matching pairs in one pass.

#### Tier 2: Algorithmic Hint (`POST /hint/2`)
* **Thought Process**: Student knows they need O(1) lookups but needs the exact logical sequence of hash map tracking.
* **Hint**: Initialize an empty dictionary. Loop through the array. For each number, calculate `complement = target - num`. If the complement is in your dictionary, return its index and the current index. Otherwise, store the current number and its index in the dictionary.
* **Conceptual Explanation**: A hash map maps values to indices. Storing elements dynamic-pass avoids checking previously visited pairs.

#### Tier 3: Code Direction Hint (`POST /hint/3`)
* **Thought Process**: Student is ready to write the O(N) solution but needs help mapping the syntax in Python.
* **Hint**: Keep it simple. Use a dictionary lookup. 
  ```python
  seen = {}
  # inside loop:
  if complement in seen:
      return [seen[complement], i]
  seen[num] = i
  ```
* **Conceptual Explanation**: Dictionary search in Python uses `in` which runs in O(1) time on average.

## Example 2: Valid Parentheses (Easy)

### 1. Student Request
* **Problem**: Given a string `s` containing just the characters `'('`, `')'`, `'{'`, `'}'`, `'['` and `']'`, determine if the input string is valid.
* **Language**: Python
* **Student's Code**:
```python
def isValid(s: str) -> bool:
    # Just counting characters
    return s.count('(') == s.count(')') and s.count('[') == s.count(']') and s.count('{') == s.count('}')
```

### 2. Progressive Hint Responses

#### Tier 1: Conceptual Hint (`POST /hint/1`)
* **Thought Process**: Student thinks simply matching counts is enough. They miss the ordering constraint (e.g. `([)]` is invalid but counts match). They need a LIFO data structure.
* **Hint**: Counting characters ignores bracket order. A string like `([)]` has equal counts but is invalid. You must track which bracket was opened last. Use a Last-In, First-Out (LIFO) data structure.
* **Conceptual Explanation**: A Stack tracks nested structures by enforcing that the last opened bracket must be the first closed.

#### Tier 2: Algorithmic Hint (`POST /hint/2`)
* **Thought Process**: Student needs the stack processing algorithm step-by-step.
* **Hint**: Iterate through the string character by character. When you see an opening bracket, push it onto a stack. When you see a closing bracket, check if the stack is empty or if the top of the stack does not match. If it mismatch, return False. Pop from stack on a match. After the loop, the stack must be empty.
* **Conceptual Explanation**: Pushing opens and popping matching closes ensures nested brackets resolve in the correct sequence.

#### Tier 3: Code Direction Hint (`POST /hint/3`)
* **Thought Process**: Student needs python list stack syntax and matching helper.
* **Hint**: Use a Python list as a stack. Map matching pairs using a dictionary for fast lookups.
  ```python
  stack = []
  mapping = {')': '(', '}': '{', ']': '['}
  # When encountering a closing bracket:
  if not stack or stack.pop() != mapping[char]:
      return False
  ```
* **Conceptual Explanation**: Lists in Python implement stack behavior using `.append()` for push and `.pop()` for pop in O(1) time.

## Example 3: Reverse Linked List (Easy)

### 1. Student Request
* **Problem**: Given the head of a singly linked list, reverse the list, and return its head.
* **Language**: Python
* **Student's Code**:
```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def reverseList(head: ListNode | None) -> ListNode | None:
    curr = head
    while curr:
        curr.next = prev  # prev is not defined!
        curr = curr.next  # Infinite loop / crash
```

### 2. Progressive Hint Responses

#### Tier 1: Conceptual Hint (`POST /hint/1`)
* **Thought Process**: Student is trying to change pointer directions in-place but has undefined variable `prev` and is losing the reference to the next node when changing `curr.next`. They need an iterative three-pointer strategy.
* **Hint**: Reassigning `curr.next` breaks your link to the rest of the list. You lose your way forward. You need to keep track of three nodes at once: the previous node, the current node, and the next node.
* **Conceptual Explanation**: In-place reversal of a singly linked list requires a temporary variable to cache the forward pointer before updating the active link.

#### Tier 2: Algorithmic Hint (`POST /hint/2`)
* **Thought Process**: Student needs the iterative node reassignment sequence.
* **Hint**: Initialize `prev` to `None`. In each step of your loop, save `next_node = curr.next`. Then reverse the link: `curr.next = prev`. Move `prev` forward to `curr`. Finally, move `curr` forward to your cached `next_node`.
* **Conceptual Explanation**: Caching the next node allows you to redirect the current pointer backward without breaking the traversal loop.

#### Tier 3: Code Direction Hint (`POST /hint/3`)
* **Thought Process**: Student needs the pointer assignment loop syntax in Python.
* **Hint**: Order of operations is critical. Run these 4 assignments inside your loop:
  ```python
  next_node = curr.next
  curr.next = prev
  prev = curr
  curr = next_node
  # Finally, return prev (the new head)
  ```
* **Conceptual Explanation**: Caching the forward path prevents infinite loops and ensures the list head transitions cleanly.
